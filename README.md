# SocialGraph — 基于社会关系网络的人物对话模拟

> **从一份聊天记录出发，构建整个社交网络，再让网络反哺对话——让 AI 不只是模仿一个人，而是理解她所处的整个世界。**

## 起源

这个项目始于一个看似 "不正经" 的尝试：把某个人的微信聊天记录蒸馏成一个 AI Skill，让 Claude 模仿她的语气说话。它确实做到了——上万条消息训练出的对话模型，能在统计约束下骗过认识她的人。

但很快就碰到了天花板：**当对话中提到另一个人时，AI 只知道硬编码的事实标签，却不知道她和他互动过多少次、在哪个群里认识的、关系是什么强度、情绪是什么色调。**

问题不在于数据不够——问题在于数据是孤立的。一份聊天记录只包含两个人的视角，而人是活在关系里的。

## 核心洞察

**一个人的语言风格不只是统计学特征——更是她的社会位置的函数。**

她在不同人面前用不同的语气。她和室友聊到凌晨三点，和班主任说话不超过五个字。她在项目群里吐槽领导，在班级群里只发"收到"。

如果不知道她在社会网络中的位置，就不可能真正拟合她的对话。

## 解决方案

SocialGraph 把 **数十份聊天记录**（私聊 + 群聊）构建成一个 **数百人、数万条消息的 Neo4j 社会关系网络**，然后用这个网络来增强对话仿真：

```
                    ┌──────────────────┐
                    │  社会关系网络      │
                    │  N 人 · M 条消息  │
                    │  PageRank · Louvain│
                    └────────┬─────────┘
                             │
    ┌────────────────────────┼────────────────────────┐
    │                        │                        │
    ▼                        ▼                        ▼
你提到同学A               你提到同学B              你提到一个陌生人
（关系紧密，常吐槽）        （仅群聊共现）             
    │                        │                        │
    ▼                        ▼                        ▼
查网络: 强度85              查网络: 强度35            查网络: 无
共同群: 3个                 共同群: 1个               
互动: 500+条                互动: 5条                 
    │                        │                        │
    ▼                        ▼                        ▼
回复: "他啊 烦得很"        回复: "哦 好像见过"       回复: "不认识"
（冷淡吐槽，匹配高强度）     （模糊，匹配低强度）       （诚实，而非瞎编）
```

**三层信息融合**：

| 层 | 来源 | 回答什么 |
|------|------|---------|
| 语义记忆 | Neo4j 关键词搜索 | "她说过什么" |
| 社会网络 | PageRank + 关系强度 + 共同群聊 | "她和谁什么关系" |
| 统计约束 | 消息行为分布 | "她怎么说话" |

## 与原始 "单人 Skill" 的区别

| | 单人 Skill v1 | SocialGraph |
|------|-------------|-------------|
| 数据源 | 1 份私聊 | **数十份聊天** (私聊 + 群聊) |
| 知识表示 | 静态 Markdown 列表 | **Neo4j 图数据库** |
| 人物认知 | 硬编码标签 | **动态关系查询** (强度/共同群/互动历史) |
| 仿真方式 | 单人物统计拟合 | **网络感知**：关系强度驱动语气变化 |
| 可扩展性 | 每增加一个人需重写配置 | **自动解析 → 自动建模** |

## 快速开始

```bash
pip install neo4j openai flask
export DEEPSEEK_API_KEY=sk-...

# 一键重建
python tools/rebuild_all.py

# 或分步
python tools/chat_parser.py --all --dir 聊天记录/texts/
python tools/neo4j_loader.py --file all_merged.jsonl --password <pwd>
python tools/batch_enrich_all.py --all
python tools/identity_resolver.py
python tools/relationship_analyzer.py
python tools/graph_analytics.py --all
python tools/temporal_evolution.py
```

## 分析能力

| 分析 | 工具 | 产出 |
|------|------|------|
| 社交中心度 | GDS PageRank | 谁是这个网络的核心 |
| 社区发现 | GDS Louvain | 自然形成的社交圈子 |
| 关系强度 | 对数缩放 | 差异化的强度谱系 |
| 时序演化 | 月度聚合 | 关系断裂的精确时间点 |
| 实体去重 | 规则+图+LLM | 跨聊天人物合并 |

## 对话仿真 API

```python
# graph_api.py 提供两个端点

# Step 0: 语义记忆查询
GET /api/graph/smart?query=你喜欢吃什么

# Step 0.5: 社会网络上下文
GET /api/graph/social?person=某人
# → {关系强度: 85, 共同群聊: [...], Identity: 规范名 ← [别名列表]}
```

部署为 Claude Code Skill 后，每次对话自动查询这两个端点，将语义记忆和社会关系上下文融合进回复生成。
